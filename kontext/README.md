# kontext/ — das Sitzungsgedächtnis

**Context retention is not claim continuity — aber Kontinuität braucht beides.**

Die Arbeit (Claims, Code, Entscheidungen) lebt in den Repos und in Jonis Ledger. Was dort
NICHT hineinpasst, ist das Episodische: der Frame, die Beweggründe, der Ton, die
unausgesprochenen Regeln der Zusammenarbeit. Jede neue Session — egal welches Modell —
startet mit leerem Kontextfenster; dieses Verzeichnis ist ihr Startkapital.

## Regeln

1. **Am Sessionanfang lesen**: das jüngste `*_uebergabe.md` vollständig, ältere bei Bedarf.
2. **Am Sessionende schreiben**: eine neue Übergabe (Datum + Modell im Dateinamen). Die
   alte wird nie editiert — append-only, wie alles hier.
3. **Ehrlich schreiben**: auch was schiefging, was offen blieb, was der Betreiber
   zurückgewiesen hat. Eine geschönte Übergabe ist wertlos.
4. Der Prototyp im übrigen Repo (`intuition_emulator/`, ehrliches NO-GO nach 7
   Durchläufen) bleibt unangetastet — abgeschlossene negative Ergebnisse sind Ergebnisse.
